Research

In this part we will focus on how the project begin to take shape from the research and projects reviews conducted by us. These include detailed analysis on different approaches to constructing an application, the programming languages and frameworks suitable for doing so, and most importantly the different explainer modules and libraries that may serve as the core of our prototype application.

Projects Review

Unfortunately there is no similar project on the subject of explainable AI to review. In general, interpretable machine learning modules come in two forms: cloud computation services with built in explainer functions, or SDK libraries that can only be imported in source codes.

However, we did find useful information and features from two products loosely related to our project:

explAIner

explAIner is a framework for explainable AI and Interactive machine learning that users can use to incorporate explainer with trained models. Though an AI explainer service itself, explAIner provided a good picture of how machine learning models work and connect with explainer tools.

Azure Machine Learning

Azure machine learning allows data scientists and developers to build hight quality machine learning models quickly and efficiently[1]. In Azure Machine Learning's "Automated ML" tool, a user selects a dataset and sets their prediction, then runs the module on a remote compute target, and Automated ML will automatically find the best algorithm for training and explaining a model to predict it.

The built-in explainability module displays explanations and relating graphs intuitively. Below is an explanation on a trained model to predict the total amount of alcohol consumed annually by countries around the world:

As seen, out of the three types of alcohols served in every country, the amount of beer served annually contributes the most to a nation's annual consumption of all alcohol, while wine and other spirits have only about half the contribution, meaning that the model's prediction is affected more by the amount of beer a country serves.

Possible Programming Languages, Frameworks and Explainer Algorithms

When choosing programming languages, we found that only Python has good support to using explainer modules and libraries. In fact, most of the explainer algorithms mentioned below can be used as Python packages. Similar libraries do exist for other programming languages, but are rare. We eventually decided to not risk losing time in trial and error with other programming languages, and use Python to code the explainer module.

There are a number of different explainer algorithms that can be the solution to our prototype product:

Summary

For possible solution and devices, we decided to make our prototype application a web-based application. By doing so the application will meet the requirements of our client: easily deployable and centred around Microsoft technologies.

For possible explainer modules, we decided to use Interpret Community SDK over other libraries. The main reasons are:

  1. Tight integration with Azure platform and other services that are used for the project, saving time and effort.
  2. Contains SHAP algorithms - SHAP offers fast computations on widely popular models like Deep Neural Networks and tree-based models, and is especially strong in precisely explaining individual predictions using Shapley values. Despite the possibility to be misused and generate misleading predictions, this has to be intentionally done on specific models, which the team cannot as this is a general purpose explainer application.
