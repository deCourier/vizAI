System Design

This system architecture diagram shows how application components interact with each other and with Microsoft Azure's services.

The Anvil Web App is the backend containing the explainability module, which is embedded in the Azure Web App. The Python server code runs on a virtual machine server hosted by Microsoft Azure. It uses Anvil’s uplink API to run the Virtual Machine as a server to load input files and to run the SHAP algorithms. It then runs the graph generation functions and creates a dashboard as an IPython HTML File running on port 80, the Virtual Machine's open HTTP port.

The Azure Web App is the frontend that serves as a user interface. Users can upload models and datasets from the embedded upload page on the Azure frontend. It is also connected to an Azure SQL Database service for data storage.

The Model Explanation Dashboard is where users see the explanations. It is embedded as a webpage into the Azure Web App frontend. 

In summary, the following functions are designed for implementation in the prototype of the product:

  1. Upload models and datasets
  2. Load a variety of possible model and dataset filetypes
  3. Apply explanation functions dynamically to any uploaded explainable model and tabular dataset
  4. Upload custom feature names for the dataset
  5. Load several explanation dashboards simultaneously
