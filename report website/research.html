Related Projects Review

Unfortunately there is no similar project to the VizAI project. In general, interpretable machine learning modules come in two forms: cloud computation services with built in explainability modules, and SDK libraries that can only be imported into source codes.

However, the team did manage to find useful information and features from Microsoft Azure's machine learning platform that helped to push the project forward:

NameAzure Machine Learning
Main features (explained in details below)Automated ML, Explainability module
What can be learnedThe importance of intuitive and logical graph presentation when explaining a machine learning model
Automated ML

Automated ML allows for data scientists, analysts, and developers to build ML models with high scale, efficiency, and productivity all while sustaining model quality[1]. In Azure Machine Learning's "Automated ML" section, a user selects a dataset and the desired column of values to predict, then starts running the module on a remote compute target, and Automated ML module will automatically find the best algorithm for training the output model and explain it.

Explainability Module

The explainability module is able to display the explanation and relating graphs intuitively. Below is an explanation on a trained model to predict the total amount of alcohol consumed annually by countries around the world:

As seen, out of the three types of alcohols served for each country, the amount of beer served annually contributes the most to a nation's annual consumption of all alcohol, while each of wine and other spirits have only about half the contribution.

From here it can be easily deduced that when predicting the annual consumption of alcohol of a country, the model's prediction is affected more by the amount of beer that country serves, making the prediction pattern logical and understandable.

Possible Programming Languages, Frameworks and Libraries for Explainability Module

Out of all available programming languages, only Python is capable of efficiently developing the explainability module. In fact, most of explainability libraries mentioned below are Python packages. Similar libraries do exist for other programming languages, however they are relatively obscure, meaning more time is consumed in trial and error.

There are a number of different explainability libraries that can be the solution to the VizAI project. The name in bold is the final chosen library:

Summary

When choosing the solution for the project, the team decided to choose web-based application over other solutions. The main reason is that it has a good compromise between performance and easy deployment - While other solutions may be better in terms of performance limit, they are often hard to deploy or simply in direct contradiction to the project requirements. 

When choosing the framework and library for the project, the team decided to use Azure Machine Learning Explainer over other libraries for the VizAI project. The main reasons are:

Tight integration with Azure platform - easy to integrate with cloud computing services and other utilities that are required for the webapp, saving considerable time and effort.

Great visual presentation - although not the most versatile explainability module, Azure Explainer is capable of generating logical and intuitive graphical presentations (as seen above in "Related Projects Review" section)
