Testing

Testing Strategy

Testing of the product prototype is divided into two parts: the backend and model compatibility tests are performed first, then the frontend is connected to the backend for integrated testing of the whole system. Compatibility testing across multiple platforms is performed after integration testing, with user acceptance testing coming last.

Formal unit testing on the backend code was considered, however due to the simplicity of  the backend code it is deemed unnecessary to perform such a test. Instead, the team performed a series of acceptance tests to ensure the explainability module functions correctly on as many types of tabular datasets and prediction models as possible.
